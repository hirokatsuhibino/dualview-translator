# Issueコメント投稿スキル

## 概要

実装が完了したとき、対応するGitHub Issueに操作方法と実装詳細をコメントとして投稿する。

## コメントを投稿する

```bash
gh issue comment <番号> --repo hirokatsuhibino/dualview-translator --body "$(cat <<'EOF'
## 実装詳細

### 操作方法

<操作手順をステップごとに記載>

---

### 実装の詳細

<変更内容の説明>

#### 変更ファイル

| ファイル | 変更内容 |
|---|---|
| `ファイル名` | 変更の概要 |

EOF
)"
```

## コメントのフォーマット

### 必須セクション

| セクション | 内容 |
|---|---|
| **操作方法** | ユーザーが実際に使う手順。スクリーンショットの代わりにコード例やUI説明を記載 |
| **実装の詳細** | 技術的な変更内容。設計判断の理由も含める |
| **変更ファイル** | ファイル名と変更の概要を表形式で列挙 |

### 任意セクション

| セクション | 含める場合 |
|---|---|
| **データモデル** | 新しいストレージ構造を追加した場合 |
| **コードスニペット** | 重要なロジックをコードで示したいとき |
| **既知の制限** | 対応できなかったエッジケースがある場合 |

## ルール

- 日本語で書く
- 操作方法は具体的に（「どのボタンを押すか」まで書く）
- コードスニペットは核心部分のみ（長すぎない）
